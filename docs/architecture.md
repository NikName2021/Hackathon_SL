# Техническая архитектура

Проект построен на современном стеке технологий, обеспечивающем высокую производительность и масштабируемость.

## Технологический стек

### Бэкенд (Backend)
*   **Язык:** Python 3.10+
*   **Фреймворк:** FastAPI (асинхронный)
*   **БД:** SQLAlchemy 2.0 (Async) + SQLite (для разработки) / PostgreSQL (для продакшена)
*   **Валидация:** Pydantic v2
*   **Аутентификация:** JWT (Access & Refresh tokens)

### Фронтенд (Frontend)
*   **Библиотека:** React 18
*   **Сборка:** Vite
*   **Язык:** TypeScript
*   **Стилизация:** Tailwind CSS
*   **Работа с API:** Axios + React Context для стейт-менеджмента

### Инфраструктура
*   **Контейнеризация:** Docker & Docker Compose
*   **Окружение:** Раздельные настройки для Dev и Prod сред

## Структура проекта

```text
/src                # Бэкенд приложение
  /app
    /api            # Эндпоинты (FastAPI Routers)
    /core           # Конфигурация, безопасность, константы
    /database       # Модели SQLAlchemy и сессии
    /helpers        # Вспомогательные функции (Auth, Watermarks)
    /schemas        # Схемы данных Pydantic
    /services       # Бизнес-логика (Task Service, Chat Service)
    /uploads        # Хранилище загруженных файлов
/spa                # Фронтенд приложение (Single Page Application)
  /src
    /api            # Axios инстансы и функции запросов
    /components     # Общие UI компоненты
    /context        # React Context (Auth, Global State)
    /pages          # Основные представления (Dashboards, Catalog)
/docker             # Docker-конфигурации
```

## Схема базы данных (Ключевые сущности)

*   **User:** Данные пользователей, роли, баллы, репутация.
*   **Task:** Основная информация о задаче, её статус и владелец.
*   **TaskApplication:** Заявки студентов на выполнение задач.
*   **TaskSubmission:** Результаты выполненных работ и отзывы.
*   **ChatMessage:** История переписки по конкретным задачам.
*   **Achievement:** Список доступных наград в системе.
*   **Skill:** Глобальный справочник компетенций.
*   **ActivityLog:** Полный аудит действий в системе.
