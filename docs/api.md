# Обзор API

Бэкенд предоставляет RESTful API, защищенный JWT-токенами. Большинство эндпоинтов требуют авторизации.

## Основные группы эндпоинтов

### 1. Авторизация и Пользователи (`/users`)
*   `POST /users/register`: Регистрация нового пользователя.
*   `POST /users/login`: Получение JWT-токенов.
*   `GET /users/me`: Информация о текущем пользователе.

### 2. Задачи (`/tasks`)
*   `GET /tasks/`: Список всех открытых задач (с фильтрацией и поиском).
*   `POST /tasks/`: Создание новой задачи (для Сотрудников).
*   `GET /tasks/{id}`: Детальная информация о задаче.
*   `POST /tasks/{id}/apply`: Подача заявки на выполнение.

### 3. Модерация (`/admin`)
*   `GET /admin/moderate/tasks`: Список задач, ожидающих одобрения.
*   `POST /admin/moderate/tasks/{id}/approve`: Одобрение задачи.

### 4. Геймификация (`/gamification`)
*   `GET /gamification/leaderboard`: Топ студентов по репутации.
*   `GET /gamification/stats`: Статистика уровней и достижений текущего пользователя.

### 5. Общение (`/chat`)
*   `GET /chat/{task_id}`: История сообщений по задаче.
*   `POST /chat/{task_id}`: Отправка текстового сообщения.
*   `POST /chat/{task_id}/file`: Отправка файла (с автоматической проверкой на безопасность).

### 6. Команды (`/teams`)
*   `POST /teams/create/{task_id}`: Создание команды для задачи.
*   `POST /teams/{team_id}/join`: Присоединение к команде.

## Формат ответов
Все ответы возвращаются в формате JSON. Ошибки сопровождаются понятными сообщениями и соответствующими HTTP-статусами (400, 401, 403, 404, 500).

Полная спецификация доступна через Swagger UI / OpenAPI по адресу `/docs` запущенного сервера.
